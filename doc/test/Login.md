# Login 頁面測試案例

> 對應元件：`src/pages/Login/index.jsx`

---

## 渲染測試

| # | 測試說明 | 前置條件 | 操作步驟 | 預期結果 |
|---|----------|----------|----------|----------|
| R-01 | ✅ 未登入時應渲染登入表單 | 使用者未登入（user = null） | 掛載 Login 元件 | 顯示帳號輸入框、密碼輸入框、登入按鈕 |
| R-02 | ✅ 應顯示提示帳號密碼的說明文字 | 使用者未登入 | 掛載 Login 元件 | 頁面中可見 `admin / admin123` 與 `user / user123` 字樣 |

## 功能測試 — 正常路徑

| # | 測試說明 | 前置條件 | 操作步驟 | 預期結果 |
|---|----------|----------|----------|----------|
| F-01 | ✅ 輸入正確的 admin 帳密後成功登入並導向 dashboard | login mock 回傳成功 | 輸入 admin / admin123，點擊登入 | 呼叫 `login()`，導向 `/dashboard` |
| F-02 | ✅ 登入請求進行中應顯示「登入中...」 | login mock 為 pending 狀態 | 點擊登入後立即檢查按鈕文字 | 按鈕文字變為「登入中...」且 disabled |
| F-03 | ✅ 登入成功後按鈕恢復可用 | login mock 回傳成功 | 等待登入完成 | 按鈕 disabled 解除（或已導頁） |

## 功能測試 — 錯誤路徑

| # | 測試說明 | 前置條件 | 操作步驟 | 預期結果 |
|---|----------|----------|----------|----------|
| E-01 | ✅ 輸入錯誤帳密應顯示錯誤訊息 | login mock throw `Error('帳號或密碼錯誤')` | 輸入錯誤帳密，點擊登入 | 頁面顯示「帳號或密碼錯誤」文字 |
| E-02 | ✅ 錯誤訊息在重新提交時應被清除 | 已顯示錯誤訊息 | 再次點擊登入 | 錯誤訊息先清空，再視結果顯示或不顯示 |
| E-03 | ✅ 登入失敗後按鈕應恢復可用 | login mock throw error | 等待登入失敗 | 按鈕 disabled 解除，可再次點擊 |
